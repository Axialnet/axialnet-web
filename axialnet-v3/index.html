GeoLLM

Domain-specific LLM for geotechnical engineering

GeoLLM is a Mistral 7B model fine-tuned with QLoRA on geotechnical engineering data — SPT N-values, boring logs, site investigation reports, and Indian geology classifications. It is designed for on-premise deployment in engineering firms that cannot use cloud-hosted AI for proprietary site data.

Base modelMistral 7B
Fine-tuningQLoRA (4-bit)
EmbeddingsBAAI/bge-m3
RetrievalBM25 + Dense → RRF
DeploymentOllama / GGUF (on-premise)
Target metricSPT N-value recall > 0.90
Training data~300 instruction pairs · 6 Indian geology types
Soil profile · SPT N-value by depth

LaneDisciplineNet

Lane violation detection for Indian road conditions

A two-stage computer vision system combining YOLOv8 object detection with CLRNet lane detection for real-time lane discipline analysis in Indian mixed traffic conditions. Trained on the India Driving Dataset (IDD) using distributed training across A100 GPUs on Param Shivay HPC.

METHODOLOGY

Why domain-specific
fine-tuning works.

General LLMs fail in high-stakes engineering contexts not due to lack of intelligence, but lack of precision. A 7B model fine-tuned on 300 domain-specific examples consistently outperforms GPT-4 on narrow geotechnical tasks.

01

Domain immersion

Practitioner vocabulary, real boring log PDFs, IS code references. Not Wikipedia summaries — actual engineering documents.

02

Synthetic data generation

GPT-4o bootstrapped instruction pairs from real site investigation reports. Fully-worked calculation chains across 6 Indian geology types.

03

QLoRA fine-tuning

4-bit quantized LoRA adapters on Mistral 7B. Deployable on a single consumer GPU. No cloud dependency at inference time.

04

Hybrid RAG pipeline

BM25 sparse + BAAI/bge-m3 dense retrieval fused via Reciprocal Rank Fusion. Depth-ordered context builder for site report PDFs.

We do. We are building from the domain down: deep understanding first, then the models to match it.
